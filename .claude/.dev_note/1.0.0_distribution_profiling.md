## 개요

CASDA는 결함 특성·배경을 분석해 ROI를 선택하고 합성 데이터를 생성하는 프레임워크지만, 19개 임계값을 하드코딩(linearity > 0.85, aspect_ratio > 5.0 등)하고 Severstal Steel 도메인에 종속되며 전문가 prior 기반 MATCHING_RULES에 의존한다. AROMA는 이 수동 규칙을 데이터 기반 적응형 정책으로 전환하는 것을 목표로 한다. 이를 위해 `dataset_config.json`(mvtec/visa/isp 공통)을 입력받아 형태·배경 피처 분포를 실측 분석하고, 클러스터링·공출현 학습으로 임계값 정책과 호환성 행렬을 도출하는 Stage 0 스크립트 `distribution_profiling.py`를 신규 추가한다. 기존 파이프라인 코드는 수정하지 않고 새 스크립트와 분석 산출물만 생성한다.

전체 명세: `.claude/.etc/1.1.0_aroma_distribution_profiling.md`

## 수정 내용

### 1. `scripts/distribution_profiling.py` — 신규 생성 (9단계 파이프라인)

#### 입력 인자

```
--dataset_config   dataset_config.json 경로 (기본: /content/AROMA/dataset_config.json)
--dataset_key      분석할 config key (예: visa_cashew, mvtec_bottle)
--output_dir       PROFILING_DIR 경로
--num_workers      병렬 워커 수 (기본: min(os.cpu_count() or 4, 8))
--max_images       디버그용 최대 이미지 수 (기본: 전체)
--n_gmm_components GMM 최대 성분 수 (기본: 5, BIC 기반 자동 선택)
```

`dataset_config.json`에서 읽는 필드: `image_dir`(good), `seed_dirs`/`seed_dir`(결함 시드), `domain`(mvtec/visa/isp)

#### 도메인별 마스크 자동 탐색

```
mvtec : {seed_dir}/../../ground_truth/{defect_type}/{stem}_mask.png
visa  : {seed_dir}/../mask/{stem}.png  (prepare_visa.py 선행 필요)
isp   : 마스크 없음 → SAM/Otsu fallback, mask_source: fallback 기록
```

마스크 미발견 시 SAM/Otsu fallback, `mask_source: fallback` 기록.  
`seed_dirs`(배열)·`seed_dir`(단수) 둘 다 처리.

#### Step 2 — Morphology Feature Extraction → `morphology_features.csv`

결함 인스턴스별 skimage.measure.regionprops:

```python
props = skimage.measure.regionprops(labeled_mask)[0]

aspect_ratio = props.major_axis_length / (props.minor_axis_length + 1e-6)
eccentricity = props.eccentricity        # 0(원) ~ 1(선)
solidity     = props.solidity            # area / convex_hull_area
extent       = props.extent              # area / bounding_box_area
circularity  = 4 * np.pi * props.area / (props.perimeter + 1e-6) ** 2

coords = np.argwhere(mask_patch)
cov    = np.cov(coords.T)
evals  = np.linalg.eigvalsh(cov)
linearity = 1.0 - evals.min() / (evals.max() + 1e-6)
```

CASDA 대비 추가: `circularity`, `eccentricity`. CASDA `fill_ratio` → `extent`(동일 개념).

#### Step 3 — Context Feature Extraction (64px 패치) → `context_features.csv`

good 이미지 + defect 이미지 non-mask 영역의 64px 패치별:

```python
local_variance       = float(np.var(patch_gray))
edge_density         = float(Sobel_mag.mean())
texture_entropy      = float(-sum(LBP_hist * log2(LBP_hist)))   # LBP P=8 R=1 uniform 10-bin
frequency_energy     = float(fft_mag[high_freq_mask].sum() / fft_mag.sum())  # 고주파 상위 25% 반경
orientation_consistency = float(-sum(angle_hist * log2(angle_hist)))  # 18-bin, 낮을수록 일관
```

5개 연속 값으로 저장 (이산화는 Compatibility Learning 단계에서 수행).  
출력 컬럼: `image_id`, `patch_xy`, 5개 피처.

#### Step 4 — Distribution Analysis → `distribution_analysis.json`

피처별(aspect_ratio, circularity, solidity, linearity, eccentricity) 50-bin 히스토그램 valley 탐지:

```python
counts, bin_edges = np.histogram(values, bins=50)
inverted = counts.max() - counts
peaks, _ = scipy.signal.find_peaks(inverted, prominence=counts.max() * 0.1)
n_valleys = len(peaks)

# n_valleys >= 2  → Multimodal → GMM (BIC 최소 성분 선택)
# n_valleys == 1  → Bimodal   → Otsu
# n_valleys == 0  → Unimodal  → Percentile [P25, P50, P75]
```

출력: `{policy, n_valleys, boundaries, percentiles{p10/p25/p50/p75/p90}}`

#### Step 5 — Morphology Clustering → `morphology_clusters.json`

정책별 클러스터 경계 도출 후 레이블 할당. centroid 기준 자동 annotation 후보 부여.  
출력: `{cluster_id, boundaries, centroid, n_samples, label}`

#### Step 6 — Compatibility Learning → `compatibility_matrix.json`

5개 Context Descriptor를 각 P33/P66 분위로 3-bin 이산화 → 5차원 Context Cell 정의.  
`P(context_cell | cluster) = count(cluster, cell) / count(cluster)`.

#### Step 7 — Deficit Analysis → `deficit_analysis.json`

```python
global_context = sum_c(P(context_cell | c) * P(c))
deficit[c]     = clip(global_context - P(context_cell | c), 0)
target_synthetic[c][cell] = deficit[c][cell] / deficit[c].sum()
```

#### Step 8 — Threshold Policy Output → `threshold_policies.json`

값(value)이 아닌 정책(policy) 저장:  
`{feature, distribution, policy, otsu_threshold or gmm_params, percentiles, valley_positions}`

#### Step 9 — recommended_config.yaml + analysis_report.md + figures/*.png

Stage 1b/3/6에서 바로 로드 가능한 `recommended_config.yaml` 생성.  
`matplotlib backend='Agg'`, `figures/` PNG 저장 (Colab headless).

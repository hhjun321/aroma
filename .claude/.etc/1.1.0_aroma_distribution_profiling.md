# 1.1.0 AROMA Stage 0 — Distribution Profiling 스크립트 구현 명세

---

## 개요

`1.0.0_analyze_dataset.md`의 CASDA analyze_dataset.py를 AROMA Distribution Profiling 방법론에 맞춰 개선한다.

CASDA와의 핵심 차이:

| 항목 | CASDA (1.0.0) | AROMA (1.1.0) |
|------|---------------|---------------|
| 도메인 | Severstal Steel 전용 (RLE/CSV) | dataset_config.json 기반 (mvtec/visa/isp 공통) |
| 배경 표현 | Semantic Category (Smooth/Stripe/...) | Physical Descriptor 연속 벡터 |
| 형태 분류 | 고정 임계값 규칙 (linearity > 0.85 등) | 분포 기반 클러스터링 (GMM/Otsu/Percentile) |
| 호환성 행렬 | 전문가 수동 설정 MATCHING_RULES | P(Context \| Morphology) 실측 공출현 학습 |
| 임계값 출력 | 고정 권장값 (p25, p50 등) | 도출 정책 저장 (bimodal→Otsu, multimodal→GMM) |
| 결핍 분석 | deficit = global_bg - P[c] (배경 type 기반) | Deficit = P(Context) - P(Context\|Cluster) (Physical Descriptor 공간) |

## 처리 흐름

```
1. dataset_config.json 로드 → dataset_key 엔트리 추출
   └─ image_dir (good) + seed_dirs/seed_dir (결함) + domain → 마스크 경로 자동 탐색

2. [Morphology Feature Extraction]
   결함 인스턴스별:
   - skimage.measure.regionprops → aspect_ratio, eccentricity, solidity, extent
   - circularity = 4π × area / perimeter²
   - linearity = 1 - λ_min/λ_max  (공분산 고유값)
   → morphology_features.csv

3. [Context Feature Extraction]
   각 이미지 64px 패치별 (good 이미지 + defect 이미지 non-mask 영역):
   - local_variance   : 패치 픽셀 분산
   - edge_density     : Sobel 응답 크기 합 / 패치 면적
   - texture_entropy  : LBP 히스토그램 엔트로피
   - frequency_energy : FFT 고주파(상위 25%) 성분 에너지 비율
   - orientation_consistency : 그래디언트 방향 히스토그램 엔트로피 (낮을수록 방향 일관)
   → context_features.csv

4. [Distribution Analysis — 형태 피처별]
   각 피처(aspect_ratio, circularity, solidity, linearity, eccentricity)에 대해:
   a. 히스토그램 valley 탐지 (scipy.signal.find_peaks on inverted)
   b. 분포 형태 판정:
      - valley ≥ 1  → Bimodal  → Otsu 정책
      - valley ≥ 2  → Multimodal → GMM 정책 (BIC로 n_components 자동 선택)
      - valley == 0 → Unimodal  → Percentile 정책 (P25/P50/P75)
   → distribution_analysis.json (형태 판정 + 선택된 정책 저장)

5. [Morphology Clustering]
   정책에 따라 클러스터 경계 도출:
   - Bimodal  : threshold = Otsu(feature_values)
   - Multimodal: GMM.fit(feature_matrix) → 성분 경계 BIC 최소 선택
   - Unimodal : bins = [P25, P50, P75]
   클러스터 레이블 할당 후 의미 annotation (사후 수동 또는 centroid 기준 자동)
   → morphology_clusters.json

6. [Compatibility Learning — P(Context | Morphology Cluster)]
   Context 공간 이산화:
   - 5개 Physical Descriptor → 각 피처 P33/P66 분위로 3-bin 이산화
   - 5차원 bin 조합 = Context Cell
   결함 인스턴스별 (cluster_id, context_cell) 공출현 집계:
   - P(context_cell | cluster) = count(cluster, cell) / count(cluster)
   → compatibility_matrix.json

7. [Deficit Analysis]
   global_context = Σ_cluster P(context_cell | cluster) × P(cluster)
   for each cluster c:
       deficit[c] = clip(global_context - P(context_cell | c), 0)
       target_synthetic[c][cell] = deficit[c][cell] / deficit[c].sum()
   → deficit_analysis.json

8. [Threshold Policy Output]
   값(value)이 아닌 정책(policy) 저장:
   {
     "feature": "aspect_ratio",
     "distribution": "bimodal",
     "policy": "otsu",
     "otsu_threshold": 3.21,        ← 이 데이터셋에서 도출한 값
     "percentiles": {"p10":..., "p25":..., "p50":..., "p75":..., "p90":...},
     "valley_positions": [3.21]
   }
   → threshold_policies.json

9. [권장 설정 생성]
   → recommended_config.yaml  (Stage 1b/3/6에서 직접 로드)
```

## Morphology Feature 상세

```python
props = skimage.measure.regionprops(labeled_mask)[0]

aspect_ratio = props.major_axis_length / (props.minor_axis_length + 1e-6)
eccentricity = props.eccentricity                          # 0(원) ~ 1(선)
solidity     = props.solidity                              # area / convex_hull_area
extent       = props.extent                                # area / bounding_box_area
circularity  = 4 * np.pi * props.area / (props.perimeter + 1e-6) ** 2

# linearity: 공분산 고유값 비율
coords = np.argwhere(mask_patch)
cov    = np.cov(coords.T)
evals  = np.linalg.eigvalsh(cov)
linearity = 1.0 - evals.min() / (evals.max() + 1e-6)
```

CASDA 대비 추가: **circularity**, **eccentricity**  
CASDA fill_ratio → AROMA **extent** (regionprops 내장, 동일 개념)

## Context Feature 상세

```python
# 64px 패치 기준
def extract_context_features(patch_gray):
    local_variance = float(np.var(patch_gray))

    sobel_x = cv2.Sobel(patch_gray, cv2.CV_64F, 1, 0, ksize=3)
    sobel_y = cv2.Sobel(patch_gray, cv2.CV_64F, 0, 1, ksize=3)
    mag     = np.sqrt(sobel_x**2 + sobel_y**2)
    edge_density = float(mag.mean())

    lbp = skimage.feature.local_binary_pattern(patch_gray, P=8, R=1, method='uniform')
    hist, _ = np.histogram(lbp, bins=10, density=True)
    hist   += 1e-10
    texture_entropy = float(-np.sum(hist * np.log2(hist)))

    fft_mag = np.abs(np.fft.fft2(patch_gray))
    fft_mag = np.fft.fftshift(fft_mag)
    h, w    = fft_mag.shape
    cy, cx  = h // 2, w // 2
    radius  = min(h, w) // 4                # 상위 25% 고주파 반경
    mask_lf = np.zeros_like(fft_mag, dtype=bool)
    Y, X    = np.ogrid[:h, :w]
    mask_lf[(Y-cy)**2 + (X-cx)**2 <= radius**2] = True
    frequency_energy = float(fft_mag[~mask_lf].sum() / (fft_mag.sum() + 1e-10))

    angles = np.arctan2(sobel_y, sobel_x)
    hist_a, _ = np.histogram(angles, bins=18, range=(-np.pi, np.pi), density=True)
    hist_a   += 1e-10
    orientation_consistency = float(-np.sum(hist_a * np.log2(hist_a)))  # 낮을수록 일관

    return {
        "local_variance": local_variance,
        "edge_density": edge_density,
        "texture_entropy": texture_entropy,
        "frequency_energy": frequency_energy,
        "orientation_consistency": orientation_consistency,
    }
```

CASDA BackgroundAnalyzer와의 차이:
- CASDA: variance/edge_strength → `Smooth/Stripe/Complex` 등 레이블 결정 (이산)
- AROMA: 5개 연속 값으로 저장 → 이산화는 Compatibility Learning 단계에서 수행

## Distribution Analysis 로직

```python
def analyze_distribution(values: np.ndarray, n_gmm_max: int = 5):
    counts, bin_edges = np.histogram(values, bins=50)
    inverted = counts.max() - counts
    peaks, props = scipy.signal.find_peaks(inverted, prominence=counts.max() * 0.1)
    n_valleys = len(peaks)

    if n_valleys >= 2:
        # Multimodal: GMM, BIC 최소 성분 선택
        policy = "gmm"
        best_bic, best_n = np.inf, 2
        for n in range(2, n_gmm_max + 1):
            gmm = GaussianMixture(n_components=n, random_state=42)
            gmm.fit(values.reshape(-1, 1))
            bic = gmm.bic(values.reshape(-1, 1))
            if bic < best_bic:
                best_bic, best_n = bic, n
        gmm_final = GaussianMixture(n_components=best_n, random_state=42).fit(values.reshape(-1, 1))
        boundaries = sorted(gmm_final.means_.flatten())

    elif n_valleys == 1:
        # Bimodal: Otsu
        policy = "otsu"
        threshold, _ = cv2.threshold(
            (values * 255 / values.max()).astype(np.uint8), 0, 255,
            cv2.THRESH_BINARY + cv2.THRESH_OTSU
        )
        boundaries = [threshold / 255 * values.max()]

    else:
        # Unimodal: Percentile
        policy = "percentile"
        boundaries = [np.percentile(values, 25),
                      np.percentile(values, 50),
                      np.percentile(values, 75)]

    return {
        "policy": policy,
        "n_valleys": int(n_valleys),
        "boundaries": [float(b) for b in boundaries],
        "percentiles": {
            f"p{p}": float(np.percentile(values, p))
            for p in [10, 25, 50, 75, 90]
        }
    }
```

## CASDA MATCHING_RULES 교체 전략

CASDA의 수동 표 (Stage 3에서 사용):

```
linear_scratch + Stripe Background = 1.0
compact_blob   + Smooth Background = 1.0
irregular      + Complex Background = 1.0
```

AROMA 교체 방법:

```python
# compatibility_matrix.json 로드
compat = json.load(open("compatibility_matrix.json"))

# 기존 MATCHING_RULES[subtype][bg_type] 접근 방식 대신:
# cluster_id + context_cell → P(context_cell | cluster)
score = compat[cluster_id][context_cell_key]
```

hybrid 옵션 (데이터 부족 시 폴백):

```python
# recommended_config.yaml: source: hybrid
score = 0.4 * manual_prior[subtype][bg_semantic]
      + 0.6 * compat[cluster_id][context_cell_key]
# 데이터가 충분하면 source: data_driven → manual_prior 가중치 0
```

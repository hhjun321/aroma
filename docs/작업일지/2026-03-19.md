## 1. Background Type 재설계 (브레인스토밍 → 스펙 → 구현)

### 배경

기존 background 분류 체계(`smooth | textured | vertical_stripe | horizontal_stripe | complex_pattern`)는 도메인별 의미가 달라 ISP-AD, MVTec AD, VisA 3개 데이터셋을 동시에 다루기 어려운 구조였다. 예를 들어 `vertical_stripe`는 ISP-AD에서는 회로 배선, MVTec에서는 목재 결, VisA에서는 볼트 나사선을 의미한다.

### 결정 사항

| 항목 | 변경 전 | 변경 후 |
|---|---|---|
| 분류 체계 | 5종 도메인 특화 | 5종 구조 기반 범용 |
| 타입명 | `vertical_stripe`, `horizontal_stripe`, `textured` | `directional`, `directional`, `organic` |
| 탐지 알고리즘 | variance → Sobel → FFT | variance → gradient entropy → autocorrelation → LBP → complex |
| 매칭 룰 테이블 | 도메인별 별도 테이블 | 단일 통합 테이블 |
| dominant_angle | 없음 | directional 타입에만 float (0–180°), 나머지 None |

### 새 분류 체계

| 타입 | 정의 |
|---|---|
| `smooth` | 균일한 단색 영역 |
| `directional` | 일관된 방향성 패턴 (선, 줄무늬, 결) |
| `periodic` | 반복 단위 패턴 (격자, 점 배열) |
| `organic` | 비방향성 자연/랜덤 텍스처 |
| `complex` | 위 2개 이상 공존 |

### 통합 매칭 룰 테이블

| Defect Subtype | smooth | directional | periodic | organic | complex |
|---|---|---|---|---|---|
| linear_scratch | 0.5 | **1.0** | 0.7 | 0.3 | 0.3 |
| elongated | 0.6 | **0.9** | 0.7 | 0.4 | 0.4 |
| compact_blob | **0.9** | 0.4 | 0.7 | 0.6 | 0.5 |
| irregular | 0.5 | 0.4 | 0.5 | **0.8** | **0.9** |
| general | 0.7 | 0.7 | 0.7 | 0.7 | 0.7 |

## 2. 데이터셋 확장 — VisA 추가

| 데이터셋 | 도메인 | 카테고리 수 |
|---|---|---|
| ISP-AD | 스크린 인쇄 | 3개 모달리티 (ASM, LSM-1, LSM-2) |
| MVTec AD | 산업 객체 | 15개 카테고리 |
| **VisA** (신규) | 혼합 산업+식품 | 12개 카테고리 |

**VisA 선택 이유:** periodic (격자 패턴 — macaroni, PCB via 배열) 타입과 organic (cashew, candle) 타입을 포함하여 기존 2개 데이터셋이 커버하지 못하던 background 구조를 추가로 검증 가능. KolektorSDD는 ISP-AD와 도메인 중복이 심해 제외.

## 3. Workers 병렬 처리 인자 추가

모든 Stage CLI에 `--workers` 인자 추가 (CASDA 패턴 기준).

```
0 = 순차 처리 (기본값)
-1 = CPU 코어 수 자동 감지 (cpu_count - 1)
N ≥ 2 = N개 프로세스 병렬 처리
```

| Stage | 병렬 단위 |
|---|---|
| stage1 | 이미지 1장 |
| stage1b | 단일 seed (N/A — workers 인자는 수용하되 무시) |
| stage2 | variant 1개 |
| stage3 | 이미지 × seed 조합 1개 |
| stage4 | 이미지 1장 |

**신규 유틸리티:** `utils/parallel.py` — `resolve_workers()`, `run_parallel()` (ProcessPoolExecutor + as_completed + tqdm)

### 주요 기술 결정 사항

- **gradient entropy vs autocorrelation 순서:** 줄무늬 패턴은 autocorrelation peak (≈1.0)와 낮은 gradient entropy 모두 발생. autocorrelation 먼저 검사하면 directional 패턴이 periodic으로 오분류됨 → gradient entropy를 먼저 검사하도록 순서 변경
- **direction_entropy_threshold = 1.0:** 초기값 2.0에서 tightening. 2.0에서는 periodic 패턴 (entropy ≈ 1.67)이 directional로 오분류됨
- **LBP entropy 정규화:** `density=True` 사용 시 확률밀도가 반환되어 entropy 계산 오류 → 명시적 정규화로 수정

## 5. 데이터셋 준비 작업

### VisA 디렉토리 재구성 스크립트 (`prepare_visa.py`)

VisA 원본 구조는 파일시스템에 train/test 분리가 없고 `split_csv/*.csv`로만 분할 정보 제공. AROMA Stage 1은 `--image_dir` 내 `*.png` 파일을 읽으므로 재구성 필요.

```bash
# VisA 재구성 (JPG 이미지인 경우 --convert_png, 병렬처리 -1=auto)
python prepare_visa.py \
    --visa_dir /content/drive/MyDrive/data/Aroma/visa \
    --convert_png \
    --workers -1
```

결과 구조:
```
visa/<category>/
├── train/good/          ← Stage 1 --image_dir
├── test/good/
├── test/anomaly/        ← Stage 1b seed 후보
└── ground_truth/anomaly/
```

**실제 VisA CSV 구조 확인 결과 (2026-03-19):**

| 항목 | 기대 | 실제 |
|---|---|---|
| CSV 파일 구성 | 카테고리별 `candle.csv`, `capsules.csv` … | 통합 `1cls.csv`, `2cls_fewshot.csv`, `2cls_highshot.csv` |
| 이미지 경로 컬럼명 | `image_path` | `image` |
| 카테고리 식별 | CSV 파일명 | `object` 컬럼 |
| 이미지 포맷 | PNG | JPG (경로: `candle/Data/Images/Normal/0836.JPG`) |

이에 따라 `prepare_visa.py`를 전면 수정:

- `reorganize_all()` → `1cls.csv` 존재 시 우선 사용; `object` 컬럼으로 카테고리 그루핑
- `_process_rows()` → `image` / `image_path`, `mask` / `mask_path` 컬럼명 양쪽 지원
- `_process_category_task()` → 모듈 레벨 래퍼 (ProcessPoolExecutor pickle 호환)
- `--workers` 인자 추가 → 카테고리 단위 병렬처리 (`run_parallel` 적용)

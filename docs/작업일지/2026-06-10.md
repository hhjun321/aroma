## 6. valley 과감지 버그 수정 — 히스토그램 bins 동적 계산

**파일**: `scripts/distribution_profiling.py`

### 원인 분석

`_detect_valleys()`의 `HISTOGRAM_BINS=50` 고정값이 92-100개 샘플에서 평균 1.84 샘플/bin 생성. 희소 bin이 inverted histogram에서 잡음 peak로 감지되어 가짜 valley 대량 발생:
- mvtec_cable circularity: 12개 (예상 ~1개)
- visa_cashew eccentricity: 11개 (예상 ~3개)
- isp_LSM_1 extent: 7개 (예상 ~3개)

### 수정 내용

`HISTOGRAM_BINS=50` → `MAX_HISTOGRAM_BINS=50` (상한 fallback 역할로 의미 변경)

bins 계산을 Sturges' rule 기반 동적 계산으로 교체:

```python
# 변경 전
bins = min(HISTOGRAM_BINS, max(len(work) - 1, 2))

# 변경 후
bins = max(int(np.ceil(np.log2(len(work)))) + 1, 5)
bins = min(bins, min(MAX_HISTOGRAM_BINS, len(work) - 1))
```

92샘플 기준: bins 50 → 8 (평균 ~11.5 샘플/bin). 잡음 peak 제거 예상.

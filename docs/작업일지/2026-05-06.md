## 셀 4 Stage 5 pending 버그 수정

### 문제
Colab에서 셀 4(Stage 5) 실행 시 `quality_scores.json`이 일부 생성된 후 스크립트가 무한 pending.

### 원인
`ThreadPoolExecutor(SEED_THREADS=4)` 내부에서 `run_quality_scoring(workers=-1)` 호출
→ 각 스레드가 `ProcessPoolExecutor(N)` 추가 생성 (중첩 병렬)
→ Linux `fork` 시 부모 프로세스의 다른 스레드가 보유한 C 레벨 락(OpenCV 등)이
  자식 프로세스에 잠긴 채 복사됨 → deadlock.

일부 `quality_scores.json`이 생성되는 이유:
ProcessPoolExecutor 진입 전(소량 이미지 seed)은 순차 처리로 완료,
나머지 seed에서 fork deadlock으로 영구 pending.

### 수정

**`docs/작업일지/phase2_execute_oneclick.md`**
- 셀 1: `IMG_WORKERS = -1` → `IMG_WORKERS = 0` (이미지 단위 순차 처리)
- 셀 4: `_run_s5` 함수 인자로 `_stage4=drive_stage4` 추가 (클로저 명시적 값 캡처)
- 셀 4: progress 출력에 처리 seed 수 추가

### 동작 변경
- 이전: `ThreadPoolExecutor(4)` × `ProcessPoolExecutor(N)` = 중첩 병렬 → deadlock
- 이후: `ThreadPoolExecutor(4)` × 순차 이미지 처리 = 단일 레벨 병렬 → 안전

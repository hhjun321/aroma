## 셀 5 Stage 6 로그 개선 + 성능 최적화

**로그 개선:** `_run_stage6` 함수 내 1~6단계 각각 시작 시 `[cat_name] N/6 ...` 형식 로그 추가.
카테고리 2개 병렬 실행 중에도 `[cat_name]` prefix로 어느 카테고리의 어느 단계인지 식별 가능.

**성능 최적화 2건:**
1. **3단계 good 이미지 복사 병렬화**: 기존 defect PNG 복사 후 good 순차 복사 →
   `ThreadPoolExecutor(max_workers=NUM_IO_THREADS+1)` 내에서 `good_fut = ex.submit(_copy_good)` 로
   defect PNG 복사와 동시 실행.
2. **6단계 Drive 업로드 병렬화**: 기존 `aroma_diffusion`, `aroma_ratio_10/20/30/50` 순차 업로드 →
   `ThreadPoolExecutor(max_workers=len(grp_list))` 로 5개 그룹 동시 업로드.
   Drive FUSE 동시 쓰기 제한이 없으므로 효과적.

**완료 로그 개선:** `[bottle] 완료 [60s] 복사:10s 구성:30s 업로드:20s diffusion=50 ratio={...}` 형식.

## seed_ratio 비율 기반 seed 샘플링 구현

`configs/benchmark_experiment_phase2.yaml` `synthesis.seed_ratio: 0.25` 설정값을
`phase2_execute_oneclick.md` 셀 3에 반영.

### 변경 내용

**셀 1 (`docs/작업일지/phase2_execute_oneclick.md`)**
- `MAX_SEEDS` (고정 수) → `SEED_RATIO` (비율) 로 변수명 및 YAML 키 변경
  ```python
  SEED_RATIO = BENCH_CFG.get("synthesis", {}).get("seed_ratio")  # None → 전체
  ```

**셀 3 — 외부 skip-check 루프**
- 전체 seed 목록 먼저 수집 → SEED_RATIO 적용 → 미완료 필터 순서로 변경
  ```python
  _all_seeds = [d for d in sorted(stage3_dir.iterdir())
                if d.is_dir() and (d / "placement_map.json").exists()]
  if SEED_RATIO is not None:
      _k = max(1, int(len(_all_seeds) * SEED_RATIO))
      _all_seeds = _random.Random(DIFFUSION_SEED).sample(_all_seeds, min(_k, len(_all_seeds)))
  seeds_todo = [d for d in _all_seeds
                if not any((drive_stage4 / d.name / "defect").glob("*.png"))]
  ```

**셀 3 — 내부 합성 루프**
- 동일한 SEED_RATIO + DIFFUSION_SEED 로 seed 목록 고정 → skip 체크 후 `seed_pm_pairs` 생성
- 외부/내부 루프가 동일한 랜덤 시드를 사용하므로 skip 판단과 실제 합성 대상이 일치

### 동작 원리
- `SEED_RATIO=0.25`, seed 100개 → 25개 고정 샘플링 (DIFFUSION_SEED 기준 재현 가능)
- `SEED_RATIO=None` → 전체 seed 사용 (기존 동작 유지)
- 카테고리마다 seed 수가 달라도 비율 기반으로 공정하게 샘플링

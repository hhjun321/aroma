## Stage 7 aroma_ratio 그룹 벤치마크 누락 버그 수정

### 문제
`stage7_benchmark.py`의 `run_benchmark()` 함수가 `aroma_ratio_*` 그룹을 실행하지 않음.

### 원인
`run_benchmark` 내부(line 803)에서 `groups` 파라미터 미전달 시 기본값으로:
```python
all_groups = groups or list(config["dataset_groups"].keys())
```
`config["dataset_groups"]`는 `baseline`, `aroma_mpb`, `aroma_diffusion` 3개만 포함.
`aroma_ratio_*` 그룹은 `config["aroma_ratio"].ratios`에서 동적으로 생성되는데 이 로직이 없었음.

노트북 셀 7에서 `run_benchmark()`에 `groups` 파라미터를 전달하지 않으므로 항상 3개 그룹만 실행.

### 수정
`stage7_benchmark.py:803` — `groups=None`일 때 `aroma_ratio` 설정도 포함하도록 수정:
```python
if groups is not None:
    all_groups = groups
else:
    _ar = config.get("aroma_ratio", {})
    all_groups = list(config["dataset_groups"].keys()) + (
        [f"aroma_ratio_{int(r*100)}" for r in _ar.get("ratios", [])]
        if _ar.get("enabled") else []
    )
```

## 연구 주제 적합 데이터셋 분석

### 연구 전제
AROMA는 **결함 발생 위치를 예측하여 ROI 영역에 합성 결함을 배치·증강**하는 연구.
PCB처럼 회선 연속성(위상학적 구조)이 필요한 카테고리는 연구 주제와 맞지 않음.

### 판단 기준
- ✅ 적합: 표면 결함, ROI 위치가 텍스처/형태에서 예측 가능, 위상학적 연속성 불필요
- ⚠️ 경계: 규칙 패턴 있으나 전기적/구조적 연속성 아님
- ❌ 부적합: 선·격자·핀 등 위상학적 연속성이 결함의 의미 자체를 결정

### 분류 결과

**✅ 적합 (16개)**
- ISP: ASM, LSM_1
- MVTec: bottle, capsule, hazelnut, leather, metal_nut, pill, tile, wood
- VisA: candle, capsules, cashew, chewinggum, fryum, macaroni

**⚠️ 경계 (4개)**
- MVTec: carpet (직물 결), screw (나사산), toothbrush (모 배열)
- VisA: pipe_fryum (연결부 제외 시 적합)

**❌ 부적합 (5개)**
- MVTec: cable (다심 케이블), grid (격자 직물), transistor (리드핀), zipper (이빨 연속성)
- VisA: pcb (PCB 회선)

### 결론
현재 벤치마크에 포함된 카테고리 중 부적합 5개는 결과 해석 시 별도 취급 또는 제외 권장.
연구 성능 평가는 적합 16개 카테고리를 기준으로 하는 것이 타당.

## image_best_f1 지표 추가

### 문제
`baseline` 그룹은 unsupervised feature distance (cosine similarity) 방식으로 동작 → defect=0 설계.
`ad_metrics.py`의 F1 계산이 `threshold=0.5` 고정 → cosine distance score는 분포가 달라 항상 F1=0.
`baseline` ↔ 분류기 그룹 간 F1 비교 불가.

### 수정
**`utils/ad_metrics.py`** — `image_best_f1` 추가:
precision-recall curve에서 최적 threshold의 F1 반환.
scoring 메커니즘과 무관하게 공정 비교 가능.

**`stage7_benchmark.py`** — record 저장 및 CSV fields에 `image_best_f1` 추가.

**`configs/benchmark_experiment_phase2.yaml`** — `evaluation.metrics`에 `image_best_f1` 추가.

### 비교 기준
- `image_f1` (threshold=0.5): 분류기 그룹 전용 실용 지표
- `image_best_f1` (최적 threshold): baseline 포함 전 그룹 공정 비교 지표
- `image_auroc`: threshold 독립, 항상 공정

## aroma_pruned 선택 방식: pruning_threshold → pruning_ratio (rank 기반)

### 원인 분석

`docs/quality_scores.json` (candle 샘플) 분석 결과:
- `artifact_score ≈ 0.84` (좋음), `blur_score ≈ 0.14` (MPB 블러), `quality_score ≈ 0.49`
- 기존 `pruning_threshold=0.6` 기준 → 전체 탈락 → `NoDefectImages`
- 도메인별 재보정(`visa: 0.4`)을 추가해도 다른 카테고리에서 동일 문제 재발 가능

### 근본 문제

threshold 방식은 도메인·카테고리마다 점수 분포가 달라 **도메인별 수동 캘리브레이션**이 지속적으로 필요하다. 또한 threshold 하나가 품질 선택과 수량을 동시에 결정하여 good:defect 비율 제어가 어렵다.

### 변경 내용

**`pruning_threshold` → `pruning_ratio` (0~1 rank 기반)**

```
aroma_full  : pruning_ratio=None → 전체 사용 (변경 없음)
aroma_pruned: pruning_ratio=0.5  → quality_score 기준 상위 50% 선택
```

- 도메인·카테고리 무관 동일 적용 (domain-agnostic)
- 항상 후보의 상위 X% 확보 → NoDefectImages 원천 차단
- `augmentation_ratio`와 독립: 품질(ratio) × 수량(augmentation_ratio) 분리 제어

#### 변경 파일

| 파일 | 변경 내용 |
|------|---------|
| `utils/dataset_builder.py` | `_collect_defect_paths`: threshold 필터 → rank 정렬 상위 K개. `build_dataset_groups`: `pruning_threshold`/`pruning_threshold_by_domain` 제거, `pruning_ratio=0.5` 추가. skip 로직·report dict 갱신 |
| `stage6_dataset_builder.py` | `run_dataset_builder`, `_build_parser`, `main()` 파라미터 교체. yaml에서 `pruning_ratio` 로드 |
| `configs/benchmark_experiment.yaml` | `pruning_threshold: 0.6` + `pruning_threshold_by_domain` 블록 제거 → `pruning_ratio: 0.5` |
| `docs/작업일지/phase1_execute.md` | Stage 6 셀: `PRUNING_THRESHOLD/BY_DOMAIN` → `PRUNING_RATIO = 0.5` |
| `docs/작업일지/phase2_execute.md` | 동일 |

| 파일 | 변경 |

### 동작

### Colab 사용 예 (candle aroma_pruned 만 재빌드)

## Stage 6 복사 최적화: preselected_defect_pairs

### 문제

Stage 6 셀 9에서 `stage4_output` 전체(수천 PNG)를 Drive → 로컬 SSD로 복사하는 과정이 오래 걸림.
Drive FUSE는 파일 1개당 네트워크 요청 1건이므로, 파일 수 자체가 병목.

### 개선 내용

**`utils/dataset_builder.py`**
- `build_dataset_groups()`에 `preselected_defect_pairs_full`, `preselected_defect_pairs_pruned` 파라미터 추가
- 제공 시 내부 `_collect_defect_paths()` 호출을 건너뛰고 전달된 쌍을 직접 사용

**`stage6_dataset_builder.py`**
- `run_dataset_builder()`에 동일 파라미터 추가 후 `build_dataset_groups`로 pass-through

**`docs/작업일지/smoke_test_3cat.md`** (셀 9)
1. Drive의 `quality_scores.json`만 읽어 `_collect_defect_paths()`로 선택 대상 결정
2. 선택된 PNG만 로컬 SSD로 복사 (`ThreadPoolExecutor`, max_workers=4)
3. 나머지 소용량 디렉토리(image_dir, test/good, seed_dirs)는 `parallel_copytree(max_workers=2)`
4. Drive→로컬 경로 리매핑 후 `preselected_defect_pairs_*`로 전달

### 효과

- 복사 파일 수 대폭 절감: 전체 stage4_output(예: 5,000장) → 실제 사용량(예: ~500장)
- `pruning_ratio` 이중 적용 방지: 선택 후 dataset_builder 재선택을 우회

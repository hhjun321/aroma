# 2026-04-17 작업일지

## 개선 내용: 복수 결함 유형 지원 (mvtec_bottle)

### 배경

MVTec bottle 카테고리는 broken_large / broken_small / contamination 세 가지 결함 유형을
보유하지만, 기존 pipeline은 broken_large 하나만 seed로 사용했다.
이로 인해 벤치마크 test set에 broken_small·contamination이 포함됨에도
학습 데이터에는 broken_large 합성본만 존재해 AUROC가 과대평가되었다.

### 변경 파일 요약

| 파일 | 변경 내용 |
|---|---|
| `dataset_config.json` | `mvtec_bottle.seed_dir` → `seed_dirs` 배열 (broken_large/small/contamination) |
| `docs/작업일지/stage1_6_execute.md` | 전 Stage 셀에서 `seed_dirs` 배열 지원 추가 |
| `utils/dataset_builder.py` | `_get_defect_type()` 헬퍼 + `balance_defect_types` 파라미터 |
| `stage6_dataset_builder.py` | `balance_defect_types` 파라미터 pass-through |
| `configs/benchmark_experiment.yaml` | `defect_type_balance: true` 추가 |

---

### 1. `dataset_config.json` — seed_dirs 배열

```json
"mvtec_bottle": {
  "domain": "mvtec",
  "image_dir": ".../bottle/train/good",
  "seed_dirs": [
    ".../bottle/test/broken_large",
    ".../bottle/test/broken_small",
    ".../bottle/test/contamination"
  ]
}
```

다른 카테고리는 `seed_dir` (단일 문자열) 유지 → 완전 하위 호환.

---

### 2. Stage 1b — seed_id 명명 규칙 변경

`seed_dirs` 배열이 2개 이상인 항목에 한해 seed_id를 `{defect_type}_{stem}` 형식으로 생성한다.

- 기존: `stage1b_output/000/`, `stage1b_output/001/`, …
- 변경: `stage1b_output/broken_large_000/`, `stage1b_output/broken_small_000/`, …

단일 `seed_dir` 항목(ISP·VisA·기타 MVTec)은 stem 그대로 유지 → 기존 데이터 영향 없음.

**⚠ mvtec_bottle 재실행 필요**: Stage 1b~6 출력물이 변경된 seed_id 체계에 맞춰
재생성되어야 한다. 기존 `stage1b_output/000/` 등 구 버전 디렉터리는 Stage 2~6에서
자동으로 무시된다 (sentinel 조건: 동일 seed_id 디렉터리가 있어야 skip).

---

### 3. `utils/dataset_builder.py` — balance_defect_types

`_get_defect_type(seed_id)`:
- `'broken_large_000'` → `'broken_large'`
- `'000'` (접두사 없음) → `''` (단일 유형)

`_collect_defect_paths(balance_defect_types=True)`:
- `stage4_output/{seed_id}/` 에서 seed_id 접두사로 결함 유형 식별
- `augmentation_ratio` 로 계산한 목표 수량을 유형 수로 균등 분배
  예: 209 good × 2.0 = 418, 3유형 → 유형당 139개
- 단일 유형 카테고리에는 영향 없음 (n_types=1 → 전체 할당)

`build_dataset_groups`:
- `balance_defect_types: bool = False` 파라미터 추가
- `build_report.json` 에 `balance_defect_types` 기록 → skip 조건 검사에 포함

---

### 4. Stage 6 notebook 셀

```python
BALANCE_DEFECT_TYPES = True  # seed_dirs 배열 항목의 유형별 균등 샘플링
```

`run_dataset_builder(..., balance_defect_types=BALANCE_DEFECT_TYPES)` 로 전달.

### mvtec_bottle 벤치마크 결과 (2026-04-17)

#### 1차 결과 — augmentation_ratio 미적용 (오류, 참고용)

`run_dataset_builder`에 `augmentation_ratio_full/pruned` 미전달 →
defect 전량 포함 (aroma_full=4524장, aroma_pruned=2262장, good=167장 대비 27×/13.5×).

| 모델 | baseline | aroma_full | aroma_pruned |
|---|---|---|---|
| yolo11 | 0.8515 | 0.9924 | 1.0000 |
| efficientdet_d0 | 0.9943 | 0.9758 | 0.9558 |

yolo11/aroma_pruned=1.0은 defect 과다(13.5×)로 인한 과대평가.

#### 2차 결과 — augmentation_ratio 명시 적용 (최종)

`augmentation_ratio_full=2.0`, `augmentation_ratio_pruned=1.5` 명시 전달 →
aroma_full=334장, aroma_pruned=251장 (good=167장 대비 2.0×/1.5×).
소요: 5.2분 (1차 30분 대비 6× 단축 — defect 수 감소 효과).

| 모델 | baseline | aroma_full | aroma_pruned |
|---|---|---|---|
| yolo11 | 0.8515 | 0.9671 | 0.9735 |
| efficientdet_d0 | 0.9943 | 0.9471 | 0.9246 |

**분석:**
- yolo11: 합성 데이터가 명확히 기여 (0.85 → 0.97). pruned(고품질 선택)이 full보다 소폭 높음
- efficientdet_d0: baseline이 이미 0.9943으로 포화 — 합성 데이터 추가 시 오히려 하락
- 두 모델 모두 aroma_pruned > aroma_full → 품질 필터링 효과 유효
- `augmentation_ratio` 미지정 시 과대평가 발생 확인 → Stage 6 명시적 ratio 전달 필수

### yolo11/aroma_pruned = 1.0 검증 방법

**원인 가설:**
1. test set에 특정 유형이 누락되어 분류 난이도가 낮음
2. aroma_pruned train/defect가 특정 유형에 편중됨 (균등 샘플링 미적용)
3. pruned 품질 기준(0.6)이 특정 유형을 과도하게 필터링

아래 Colab 셀로 각 가설을 확인한다.

```python
import json
from pathlib import Path

REPO      = Path("/content/aroma")
CAT_DIR   = Path("/content/drive/MyDrive/data/Aroma/mvtec/bottle")
AUG_DIR   = CAT_DIR / "augmented_dataset"
STAGE4_DIR = CAT_DIR / "stage4_output"

# ── 1) test set 구성 확인 ─────────────────────────────────────
# baseline/test 가 원본 MVTec test set을 그대로 사용하는지,
# 3개 결함 유형이 모두 포함되는지 확인
print("=== 1) test set 구성 ===")
for group in ["baseline", "aroma_full", "aroma_pruned"]:
    test_dir = AUG_DIR / group / "test"
    if not test_dir.exists():
        print(f"  {group}/test: 없음")
        continue
    is_symlink = test_dir.is_symlink()
    subdirs = sorted([d.name for d in test_dir.iterdir() if d.is_dir()])
    counts  = {d: len(list((test_dir / d).glob("*.png"))) for d in subdirs}
    print(f"  {group}/test {'[symlink]' if is_symlink else '[copy]'}: {counts}")

# ── 2) aroma_pruned train/defect 결함 유형별 구성 확인 ────────
# seed_id 접두사(broken_large_, broken_small_, contamination_)로 유형 판별
print("\n=== 2) aroma_pruned train/defect 유형별 구성 ===")
for group in ["aroma_full", "aroma_pruned"]:
    defect_dir = AUG_DIR / group / "train" / "defect"
    if not defect_dir.exists():
        print(f"  {group}: defect 디렉터리 없음")
        continue
    type_counts = {}
    for f in defect_dir.glob("*.png"):
        # 파일명: {seed_id}_{image_id}.png 형식
        parts = f.stem.split("_")
        # broken_large_000_img001 → broken_large
        if len(parts) >= 3 and parts[0] in ("broken", "contamination"):
            defect_type = "_".join(parts[:2]) if parts[0] == "broken" else parts[0]
        else:
            defect_type = "unknown"
        type_counts[defect_type] = type_counts.get(defect_type, 0) + 1
    total = sum(type_counts.values())
    print(f"  {group} (총 {total}장): {type_counts}")

# ── 3) Stage 4 seed별 quality score 분포 확인 ─────────────────
# 유형별 평균 품질 점수 — 특정 유형이 pruning으로 과도하게 제거되는지 확인
print("\n=== 3) 결함 유형별 quality score 분포 ===")
type_scores = {}
for seed_dir in sorted(STAGE4_DIR.iterdir()):
    if not seed_dir.is_dir():
        continue
    qs_path = seed_dir / "quality_scores.json"
    if not qs_path.exists():
        continue
    # seed_id에서 결함 유형 추출
    seed_id = seed_dir.name
    if seed_id.startswith("broken_large"):
        dtype = "broken_large"
    elif seed_id.startswith("broken_small"):
        dtype = "broken_small"
    elif seed_id.startswith("contamination"):
        dtype = "contamination"
    else:
        dtype = "unknown"

    scores = json.loads(qs_path.read_text())
    if isinstance(scores, list):
        vals = [s["final_score"] for s in scores if "final_score" in s]
    elif isinstance(scores, dict):
        vals = [v.get("final_score", 0) for v in scores.values() if isinstance(v, dict)]
    else:
        vals = []

    if vals:
        if dtype not in type_scores:
            type_scores[dtype] = []
        type_scores[dtype].extend(vals)

THRESHOLD = 0.6
for dtype, vals in sorted(type_scores.items()):
    above = sum(1 for v in vals if v >= THRESHOLD)
    print(f"  {dtype}: 총 {len(vals)}장, "
          f"평균={sum(vals)/len(vals):.3f}, "
          f"threshold({THRESHOLD}) 이상={above}장 ({100*above/len(vals):.0f}%)")

# ── 4) build_report.json 확인 ─────────────────────────────────
# balance_defect_types 가 실제로 적용되었는지, 유형별 할당 수량 확인
print("\n=== 4) build_report.json ===")
report_path = AUG_DIR / "build_report.json"
if report_path.exists():
    report = json.loads(report_path.read_text())
    print(f"  balance_defect_types: {report.get('balance_defect_types', 'N/A')}")
    for group in ["baseline", "aroma_full", "aroma_pruned"]:
        g = report.get(group, {})
        print(f"  {group}: good={g.get('good_count','?')}, defect={g.get('defect_count','?')}")
else:
    print("  build_report.json 없음")
```

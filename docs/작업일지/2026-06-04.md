## AROMA Stage 0 Distribution Profiling 스크립트 신규 구현

### 배경

AROMA 파이프라인의 Stage 1b/3/6에서 사용하는 형태 분류 임계값과 ROI 호환성 점수가
수동으로 하드코딩되어 있어 도메인 일반화가 어렵다. 실측 데이터 분포로부터 임계값 정책과
호환성 행렬을 도출하는 Stage 0 분석 스크립트를 신규 추가한다.

### 변경 파일

- **`scripts/distribution_profiling.py`** (신규)
- **`requirements.txt`** — `scikit-learn>=1.3.0` 추가

### 구현 내용

`scripts/distribution_profiling.py` — 9단계 파이프라인:

| 단계 | 출력 파일 | 설명 |
|------|----------|------|
| Step 1 | — | `dataset_config.json` 로드, 도메인별 마스크 경로 자동 탐색 |
| Step 2 | `morphology_features.csv` | 결함 인스턴스별 6개 형태 피처 (linearity/solidity/extent/aspect_ratio/eccentricity/circularity) |
| Step 3 | `context_features.csv` | 64px 패치별 5개 컨텍스트 피처 (local_variance/edge_density/texture_entropy/frequency_energy/orientation_consistency) |
| Step 4 | `distribution_analysis.json` | 피처별 valley 탐지 → Bimodal/Multimodal/Unimodal 판정 + 정책 선택 |
| Step 5 | `morphology_clusters.json` | GMM/BIC 기반 형태 클러스터링 |
| Step 6 | `compatibility_matrix.json` | P(context_cell \| cluster) 공출현 학습 |
| Step 7 | `deficit_analysis.json` | Deficit = global_context - P(cell\|cluster) → target_synthetic |
| Step 8 | `threshold_policies.json` | 임계값 도출 정책 저장 (값이 아닌 정책) |
| Step 9 | `recommended_config.yaml` + `analysis_report.md` + `figures/*.png` | 리포팅 |

**도메인별 마스크 탐색 전략:**
- `mvtec`: `ground_truth/{defect_type}/{stem}_mask.png` 자동 탐색
- `visa`: `test/mask/{stem}.png` (prepare_visa.py 선행 필요)
- `isp`: GT 없음 → SAM/Otsu fallback, `mask_source: fallback_*` 기록

**입력 인자:**
```
--dataset_config  dataset_config.json 경로
--dataset_key     분석할 카테고리 키 (예: visa_cashew, mvtec_bottle)
--output_dir      결과 저장 경로 (PROFILING_DIR)
--num_workers     병렬 워커 수 (-1=auto)
--max_images      디버그용 이미지 수 제한
--n_gmm_components  GMM 최대 성분 수 (기본 5, BIC 자동 선택)
```

**재사용 모듈:**
- `utils/defect_characterization.py:DefectCharacterizer` → linearity/solidity/extent/aspect_ratio
- `stage1b_seed_characterization.py:extract_seed_mask` → SAM/Otsu fallback
- `utils/parallel.py:run_parallel, resolve_workers` → 멀티프로세싱
- `utils/io.py:load_json, save_json, safe_imread`
